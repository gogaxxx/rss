Структура основого скрипта:

"Конфигуратор"
1. Получает id фида из комстроки
2. По id получает параметры: урл, каталог кеша, каталог с готовыми
	файлами
3. Передаёт структуру модулю "Скачиватель"
	Псевдокод:
		$id = $ARGV[0];
		$cfg = get_cfg_by_id($id);

"Скачиватель"
4. Получает из структуры урл и каталог кеша.
5. Проверяет, изменился ли ресурс по указаному урлу
6. Если да, скачивает ресурс и сохраняет в файл кеша
7. Заполняет поле "содержимое" в структуре и передаёт структуру модулю
"Парсер"
	Псевдокод:
		$url = $cfg->{url};
		$cache_dir = $cfg->{cache_dir};
		$cache_file = get_cache_file_name($cfg);
		if (changed($url,$cache_file)) {
			download($url, $cache_file);
		}
		$cfg->{content} = load_file($cache_file);

"Парсер"
8. В структуре его интересует только поле "содержимое"
9. Разбирает содержимое, заполняя для каждой найденной статьи структуру
"Статья" со следующими полями:
	ид
	дата
	содержимое (зависит от типа ресурса)
10. Для каждой найденной статьи вызывает модуль "Сохранятель"
	Псевдокод:
		$parser->start_parsing($cfg->{content});
		while($item = $parser->next_item()) {
			saver($cfg, $item);
		}
		$parser->finish_parsing();

"Сохранятель"
11. Получает от "Парсера" основную структуру-конфигурацию и одну
	структуру-статью
12. Производит произвольные манипуляции для подготовки статьи к
	сохранению в локальное хранилище. Например, преобразует текст, или
	скачивает дополнительные файлы.
13. Сохраняет полученную статью в локальное хранилище.
	Псевдокод:
		$cooked_item = cook($item);
		save($item->{id},
			 $item->{date},
			 $cooked_item);
